The Basics of IRIS3

Pipeline Overview

IRIS3 (Integrated Cell-type-specific Regulon Inference Server from Single-cell RNA-Seq) is an integrated web server for cell-type-specific regulon (CTSR) prediction from Human or Mouse Single-cell RNA-Seq data, which allows researchers to pinpoint crucial regulators and target genes that are involved in the development of complex diseases.

The unique features of IRIS3 include:

(1) It is an all-in-one pipeline that provides de novo identification of regulons, organically integrating biclustering for cell-type-specific gene module detection and de novo motif prediction in support of potential novel regulons discovery;

(2) It provides informative regulon interpretations with interactive visualizations, in support of in-depth analysis in studying heterogeneous regulatory mechanisms;

(3) It is equipped with a user-friendly web interface with a simple submission process and interactive analysis functionalities that requires no programming knowledge.

Input:

1. scRNA-Seq gene expression matrix (Required):
The required input is gene expression data with three acceptable formats

  1. A single txt, tsv or csv formatted gene expression matrix. The compressed format (gzip) is accepted.
  2. An hdf5 feature barcode matrix
  3. The three files recording information of barcodes, features, and gene expressions in the 10X genomics output folder.

The gene expression matrix (GEMAT) file with genes as rows and cells as columns. Both Gene Symbols (e.g. HSPA9), Ensembl Gene IDs (e.g. ENSG00000113013), or Transcription IDs (e.g.: ENSMUST00000074805) are allowed. Our server will try to match your IDs to Human and mouse databases.

2. Cell label file (Optional): a two-column matrix with the first column as the cell names exactly matching the gene expression file, and the second column as ground-truth cell clusters. The cluster indicator could either be terms (e.g. 2_cell_stage, 4_cell_stage) or numbers (e.g. 1,2). The cell label file will be used to evaluate the predicted cell types (evaluation omitted otherwise) and regulon inference (or use the predicted cell type instead).

3. Custom gene module file (Optional): a text file, each column should contain one gene list. For those users who have their own gene modules, either collected from literature or generated from other tools, those regulons will be regarded as "module-specific regulons" and have individual tabs next to the last cell type to show the heatmap and results. In addition to the similar regulon finding function described below, users can compare regulons of interest with the regulons referred from the self-uploaded gene modules and find any similarities.

The overall pipeline includes seven steps:

Our server organically integrates multiple state-of-the-art tools, including DrImpute, scran, Seurat, QUBIC2, DMINDA2, and MEME. These tools are either reputed or have been justified with the best performance in the analytical area.

Step I: Pre-processing

The gene expression data is first loaded, and a Seurat object is created. Genes with zero values in more than 0.1% cells and cells with zero values in 200 genes are removed to obtain reliable and robust analytical performances. Data normalization status is auto-detection by considering integers as non-normalized values, whereas decimals are normalized. The unnormalized data will be normalized by scran. Gene imputation is set up using DrImpute with default settings. Finally, the expression values are log-normalized to rescale the data to match its neighbors.

Step II: Cell type prediction

Cell types are predicted in Seurat (version 3.1) with most parameters set to default. The number of PCs (principal component) is set to ten as suggested by the Seurat tutorial, and the elbow plot is generated for each test dataset. Normally, the top ten PCs can cover 85%-95% of the data variation which is enough for feature selection. The output of this step is a two-column cell label that will be used in Step IV and additional trajectory analysis.

Step III: Gene module detection

The pre-processed data from Step I is analyzed by our in-house biclustering tool, QUBIC2, for gene module detection. Each of the identified biclusters represents a group of co-expressed genes under a specific subset of cells.

Two unique features are addressed in QUBIC2: (1) it can identify all statistically significant biclusters including biclusters with the so-called 'scaling patterns', a problem considered to be rather challenging; and (2) the algorithm solves such general biclustering problems very efficiently, capable of solving biclustering problems with tens-of-thousands of genes under to thousands of conditions in a few minutes.

Three major parameters control the output biclusters in IRIS3:

a. Bicluster overlap rate controls the level (from 0 to 1) of overlaps between the to-be-identified biclusters. 0 means no overlap and 1 indicates complete overlap. The default value is 0.7.

b. Maximum bicluster number controls the maximum number of to-be-identified biclusters. The default value is 500. A smaller value may sufficiently decrease the running time but may identify less regulons.

c. Minimum cell number The minimum column width of the bicluster block. The default is 20.

Step IV: Gene module assignment

We consider the component genes of a bicluster to respond to the regulatory signal in a specific cell type if the cells in the bicluster are highly consistent with the cells in the cell type. To determine the consistency, a hypergeometric enrichment test is performed using the cell components of identified biclusters from Step II and the cell types predicted from Seurat from Step III (or the uploaded ground-truth cell types). A bicluster is a so-called CTS-gene-module if its cell components are significantly consistent with a cell type (p-value < 0.05, Bonferroni adjusted). Thus, a CTS-gene module is possibly found in multiple cell types, as long as it is significantly enriched in those cell types. The output of this step is the CTS-gene-modules, if present, of each of the identified cell types, which lays a solid foundation of the regulon identification.

Users with developed gene modules that they are interested in or identified by their preferred module detection methods can also upload these modules to IRIS3 and have them analyzed to identify the module-specific regulons.

Step V: Motif finding and comparison

For each cell type, putative motifs are then identified in each gene module via de novo motif prediction function in MEME and DMINDA2. The default 1,000 bp upstream promoter sequences of each gene are extracted using the hg38/mm10 reference implemented in the R/Bioconductor library BSgenome.Hsapiens.UCSC.hg38 and BSgenome.Mmusculus.UCSC.mm10 for human and mouse data, respectively. Default parameters using the differential enrichment mode based on the enrichment of their sites in the query sequences compared with the human/mouse background sequences.

Step VI: Active regulon determination

The identified de novo motifs in a specific cell type were annotated and clustered with best matching known motifs from the HOCOMOCO database (V11) using TOMTOM. Matching motifs were filtered the removing HOCOMOCO target for those with the q-value of less than 0.05. The q-value is the minimal false discovery rate at which the observed similarity would be deemed significant. For each of the motif clusters, the corresponding nonredundant gene list is named as an active regulon.

Step VII: regulon inference

For each active regulon, the RAS in a cell is calculated using the Wilcoxon rank sum test (wmwTest function in the BioQC R package) to rank the expression value of genes in the regulon to the rest genes. This method output more regulons and also is faster than the AUCell method used in SCENIC and not dependent on selecting thresholds. The procedure of calculating the specificity of an active regulon under one cell type is adopted based on an entropy-based strategy which is previously used for gene expression data analysis. More details about the calculation of regulon specificity score can be found here.

Analysis and visualization

The predicted cell types (from Step II) can be evaluated by calculating the silhouette score that measures how similar a cell is to its type compared to other clusters.

If the ground-truth cell types are provided by users, the predicted labels will also be evaluated by the Adjusted Rand Index (ARI), Rand Index (RI), Fowlkes and Mallows's index (FMI), and Jaccard Index (JI). A Sankey plot can be generated to show the convergence and divergence of a two cell groups (ground-truth versus Seurat prediction). The width of the arrows is shown proportionally to the flow quantity. More details about the calculation of the cell type prediction evaluation can be found here.

The regulons can be sent to perform the enrichment analysis against different functional annotation databases to identify the enriched GO functionalities, biological pathways, etc. The enrichment test is performed by Enrichr. The regulon heatmap and corresponding gene expression pattern interpretation are supported by Clustergrammer, which is a powerful and user-friendly heatmap generator newly developed by the Ma'ayan Lab.
